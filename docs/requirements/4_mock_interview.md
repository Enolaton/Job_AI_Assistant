# 기능 4: AI 모의면접 (AI Mock Interview) - 요구사항 명세서 초안

## 1. 기능 개요 및 목표 (Overview & Goal)
*   **목표:** 작성된 자기소개서(`CoverLetter`)와 직무 분석(`JobAnalysis`) 내용을 바탕으로 실제 직무 면접(Technical/Fit)에서 나올 법한 생생한 질문을 AI가 던지고, 사용자가 이에 답변하며 실전 대처 능력을 기른다.
*   **핵심 가치:** 뻔한 면접 리스트가 아닌 **'내 경험'과 '내 자소서 내용'에 딱 맞춘 압박 질문 및 꼬리 질문**으로, 실제 면접관이 앞에 있는 듯한 몰입감을 제공하고 끝난 후 맞춤형 피드백 리포트를 제공한다.

## 2. 사용자 흐름 (User Flow)
1.  **AI 모의면접 설정:**
    *   어떤 공고/자소서를 기반으로 면접을 볼 것인지 타겟 선택.
    *   면접 난이도 설정 (예: 신입 기본, 실무자 압박, 임원 인성 등).
    *   질문 개수 선택 (예: 5개, 10개).
2.  **모의면접 대기실:** 카메라 및 마이크 상태 체크, 심호흡 후 '입장' 버튼 클릭.
3.  **실전 면접 진행 (인터랙션 화면):**
    *   AI 면접관의 화면(또는 아바타)이 표시되고 텍스트/음성(TTS)으로 첫 질문 투척.
    *   사용자는 음성으로 답변 (옵션: 음성을 인식(STT)하여 텍스트로 실시간 표시).
    *   답변이 끝나면 '답변 완료' 버튼을 누르거나 일정 시간 침묵 시 자동 종료.
4.  **꼬리 질문 발생 (AI 다이내믹 핑퐁):**
    *   사용자의 답변 내용에 따라 추가 설명이 필요하거나 모순된 내용이 있다면 압박용 꼬리 질문(Follow-up question) 제시. (예: "그 프로젝트에서 갈등이 있었다고 하셨는데, 왜 그 방식을 쓰지 않으셨나요?")
5.  **면접 종료 및 피드백 생성:** 모든 질문 세션 종료 후 분석 로딩.
6.  **면접 결과 리포트 (Feedback Report):**
    *   전체 평가 점수, 답변 속도/명확성 진단 (음성 인식 기반일 경우).
    *   각 질문별 내 답변 스크립트(STT 변환 텍스트)와 함께 **모범 답변(더 나은 답변 방향) 제안**.
    *   보완해야 할 핵심 경험/역량 피드백.

## 3. AI 연동 시나리오 (AI Interactions)

### 3.1. 맞춤형 질문 리스트 생성 (Pre-Interview)
*   **Trigger:** 모의면접 설정 완료 시, 백그라운드에서 면접에 사용할 동적 질문 풀(Pool) 생성.
*   **Input Data:** 직무 요건(JD) 키워드 + 사용자가 선택한 자기소개서 본문 + 내 경험 뱅크의 STAR 데이터.
*   **System Prompt (예시):**
    > "너는 [기업명]의 [직무명] 실무 면접관이야. 다음은 지원자의 자기소개서 내용이야. 지원자의 역량을 깊이 검증할 수 있는 핵심 질문 5개를 만들어줘. 일반적인 질문 2개, 자소서 내용을 꼬집는 심층 질문 3개로 구성해. 특히 지원자가 [경험 내용]에서 쓴 기술 스택이나 성과의 진위 여부를 파고드는 질문 하나를 반드시 포함해."
*   **Output Format:** JSON Array (질문 리스트 + 각 질문의 의도)

### 3.2. 실시간 꼬리 질문 생성 (During Interview)
*   **Trigger:** 사용자의 1차 답변이 짧거나, 논리적 비약이 있을 때 (답변 후 `생성` 요청 트리거).
*   **Input Data:** 면접관의 원 질문 내용, 방금 전 사용자가 대답한 STT 스크립트.
*   **System Prompt:**
    > "지원자의 방금 답변 내용은 [답변 텍스트]야. 이 답변이 불충분하거나 논리가 부족하다면, 면접관으로서 날카로운 꼬리 질문(압박 질문)을 1개만 생성해줘. 만약 답변이 충분히 좋다면 '통과' 라고 대답해."

### 3.3. 상세 피드백 리포트 생성 (Post-Interview)
*   **Trigger:** 면접 종료 직후 전체 세션 평가.
*   **Input Data:** 모든 대화 세트(질문1 - 답변1, 질문2 - 답변2...).
*   **System Prompt:**
    > "지금까지 진행된 면접 로그를 분석해서 전체적인 피드백 리포트를 써줘. (1) 지원자의 강점, (2) 가장 아쉬웠던 답변 분석과 그 이유, (3) 해당 질문에 대한 모범 답변 예시(STAR 기법 기반)를 포함해야 해."

## 4. 데이터 모델 (Data Model - DB 스키마 초안)
*   `MockInterview` (모의면접 세션 테이블)
    *   `id` (PK)
    *   `userId` (FK, 참여자)
    *   `coverLetterId` (FK, 기준 자소서)
    *   `totalScore` (Int, 전체 평가 점수)
    *   `feedbackReport` (JSON, 종합 피드백)
    *   `createdAt`
*   `InterviewQuestion` (질의응답 테이블)
    *   `mockInterviewId` (FK)
    *   `question` (String, AI가 던진 질문)
    *   `answer` (Text, 사용자의 STT 답변 또는 텍스트 답변)
    *   `isFollowUp` (Boolean, 꼬리 질문 여부)
    *   `aiEvaluate` (Text, 이 답변에 대한 개별 피드백/모범답안)

## 5. UI/UX 요구사항 (UI/UX Requirements)
*   **포커스 모드 (다크 톤):** 면접 중에는 화면 주변의 메뉴바나 방해 요소를 모두 숨기고 중앙의 타이머, 음성 파형(Waveform), 질문 텍스트만 띄워 몰입감을 극대화. 어두운 배경(Dark theme)을 기본으로 적용하여 긴장감 유지.
*   **STT/TTS 음성 인터페이스 필수 지원 여부 판단:** 텍스트 채팅형(챗봇 형태)으로 개발할지, 아니면 Web Speech API를 활용해 실제 음성 면접처럼 주고받을지 구체적 결정 필요. (비용과 개발 난이도 차이가 크기 때문)
*   **결과 리포트 아코디언 컴포넌트:** 각 질문/답변 쌍은 아코디언(열었다 닫았다 하는 컴포넌트) 형태로 만들어, 리포트 화면을 내릴 때 사용자가 자신의 실수 구간을 콕 집어 디테일하게 복기할 수 있게 제작.
