# 기능 1: 내 경험 뱅크 (My Experience Bank) - 요구사항 명세서 초안

## 1. 기능 개요 및 목표 (Overview & Goal)
*   **목표:** 사용자가 과거의 프로젝트, 인턴십, 대외활동, 학업 등의 경험을 체계적으로 기록하고 관리할 수 있도록 돕는다. 
*   **핵심 가치:** 단순히 텍스트를 저장하는 것을 넘어, **AI가 사용자의 경험을 분석하여 STAR(Situation, Task, Action, Result) 기법 등 역량 중심 서술로 구조화**하는 것을 돕고 핵심 키워드를 추출하여 추후 '자기소개서' 및 '모의면접' 기능에서 즉시 활용할 수 있는 데이터 베이스를 구축한다.

## 2. 사용자 흐름 (User Flow)
1.  **목록 확인:** '내 경험 뱅크' 페이지 진입 시 등록된 경험 목록을 타임라인 또는 카드 형태로 확인.
2.  **경험 추가:** `+ 새 경험 추가` 버튼 클릭.
3.  **기본 정보 입력:** 경험 제목, 기간, 역할, 소속/조직명 등 기본 메타데이터 입력.
4.  **상세 내용 작성 (AI 어시스턴트 활용):**
    *   사용자가 자유 형식으로 경험 내용을 작성 (또는 키워드만 나열).
    *   `✨ AI로 내용 다듬기` 버튼 클릭.
5.  **AI 피드백 및 구조화:** AI가 작성된 내용을 바탕으로 부족한 부분(예: "어떤 구체적인 성과를 냈나요?")을 질문하거나, STAR 기법에 맞게 내용을 자동으로 재구성하여 제안.
6.  **수정 및 저장:** 사용자가 AI의 제안을 수락하거나 내용을 추가 수정한 뒤 최종 저장.
7.  **태그/키워드 자동 생성:** 저장 완료 시 AI가 이 경험에서 돋보이는 핵심 역량 키워드(예: #리더십, #문제해결, #데이터분석)를 자동 추출하여 태그로 등록.

## 3. AI 연동 시나리오 (AI Interactions)

### 3.1. 경험 구조화 어시스턴트
*   **Trigger:** 사용자가 경험 내용을 작성하고 `AI로 내용 다듬기` 요청.
*   **Input Data:** 제목, 기간, 역할, 사용자가 작성한 초안 텍스트.
*   **System Prompt (예시):**
    > "너는 전문 취업 컨설턴트야. 사용자가 제공한 경험 데이터를 바탕으로 이력서와 자기소개서에 활용하기 좋은 STAR(상황, 과제, 행동, 결과) 기법 기반의 형태로 글을 다듬어줘. 만약 성과(Result)나 구체적인 행동(Action)이 명확하지 않다면 어떤 내용을 보완해야 하는지 짧은 코멘트를 남겨줘."
*   **Output Format:** JSON 구조 (구조화된 텍스트, 보완 요청 멘트)

### 3.2. 역량 키워드 추출
*   **Trigger:** 경험 최종 저장 시 백그라운드에서 동작.
*   **Input Data:** 구조화가 완료된 경험 상세 텍스트.
*   **System Prompt (예시):**
    > "제공된 경험 내용을 분석해서, 직무 역량과 관련된 핵심 키워드를 최대 5개 추출해줘."
*   **Output Format:** 배열 형태 `["리더십", "소통", "Python", "문제해결"]`

## 4. 데이터 모델 (Data Model - DB 스키마 초안)
*   `Experience` (경험 테이블)
    *   `id` (PK, UUID/Int)
    *   `userId` (FK, 작성자)
    *   `title` (String, 여정 앱 개발 프로젝트)
    *   `organization` (String, 멋쟁이사자처럼)
    *   `role` (String, 프론트엔드 리드)
    *   `startDate` (DateTime)
    *   `endDate` (DateTime, 널 허용)
    *   `isCurrent` (Boolean, 진행 중 여부)
    *   `rawContent` (Text, 사용자가 처음 쓴 원본 내용)
    *   `structuredContent` (Text, AI가 다듬거나 최종 저장된 내용)
    *   `createdAt`, `updatedAt`
*   `ExperienceTag` (태그 테이블)
    *   `experienceId` (FK)
    *   `keyword` (String, 예: "리더십")

## 5. UI/UX 요구사항 (UI/UX Requirements)
*   **경험 작성 폼:** 입력이 부담스럽지 않도록 단계별(Wizard) 입력 방식 또는 명확히 구분된 섹션 적용.
*   **AI 상호작용 피드백:** AI가 텍스트를 생성하거나 키워드를 추출할 때, 스켈레톤 로딩이나 반짝이는 애니메이션(Sparkles)을 통해 'AI가 작업 중'임을 직관적으로 안내. (Bunny의 지능형 에이전트 컨셉 강조)
*   **가독성 높은 목록:** 저장된 경험을 시간순 타임라인으로 배치하고, 각 경험 카드에는 중요 태그(키워드)가 한눈에 보이도록 디자인.
