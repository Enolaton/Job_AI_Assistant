# 기능 3: 자기소개서 작성 (Cover Letter Generator) - 요구사항 명세서 초안

## 1. 기능 개요 및 목표 (Overview & Goal)
*   **목표:** 사용자가 지원하고자 하는 직무(공고)와 자신의 경험(내 경험 뱅크) 데이터를 바탕으로 빠르고 완성도 높은 자기소개서 초안을 작성하도록 돕는다.
*   **핵심 가치:** 텅 빈 화면에서 겪는 막막함을 해소하고, AI가 지원 직무에 딱 맞는 경험을 골라주어 면접관이 매력적으로 느낄 수 있는 자기소개서를 쉽게 빌드업하는 것.

## 2. 사용자 흐름 (User Flow)
1.  **새 자기소개서 만들기 진입:**
    *   옵션 A: 직접 새 문서 열기.
    *   옵션 B: '직무 분석' 결과 화면에서 연동 버튼 클릭.
2.  **문항 입력 및 타겟 설정:**
    *   기업명, 직무명 입력 (연동 시 자동 입력).
    *   자소서 문항(예: "우리 회사에 지원한 동기와 그것을 달성하기 위한 구체적인 노력을 작성해주세요.")과 글자 수 제한 설정.
3.  **내 경험 연결 (AI 추천):**
    *   문항 내용을 분석하여 답변에 가장 적합한 '내 경험 뱅크' 데이터를 AI가 자동 추천하거나, 사용자가 체크박스로 직접 추가.
4.  **AI 초안 생성 (Magic Draft):**
    *   `✨ 초안 작성하기` 클릭 시, 선택된 경험 데이터를 바탕으로 해당 기업/직무에 맞는 자소서 초안 생성.
5.  **에디터 편집 및 윤문:**
    *   사용자가 생성된 초안을 읽으며 문구를 자연스럽게 수정하는 WYSIWYG 형태의 글쓰기 모드 지원.
    *   에디터 내에 `글자 수 확인`, `맞춤법 교정`, `문장 다듬기(어조 변경)` 유틸리티 제공.
6.  **완성 및 저장:** 초안을 최종 완성본으로 저장 (PDF, Word 내보내기 및 클립보드 복사).

## 3. AI 연동 시나리오 (AI Interactions)

### 3.1. 경험 추천 어시스턴트
*   **Trigger:** 자소서 문항을 입력했을 때 실시간 동작.
*   **Input Data:** 기업명, 직무명, 자소서 문항 텍스트 + '내 경험 뱅크'의 모든 경험 요약본(혹은 태그들).
*   **System Prompt (예시):**
    > "자소서 문항과 지원하려는 기업/직무가 주어집니다. 이 문항에 답변하기 가장 좋은 지원자의 핵심 경험을 최대 2개 추천하고, 왜 이 경험이 적합한 소재인지 이유를 설명해줘."

### 3.2. 자소서 초안 생성 (Magic Draft)
*   **Trigger:** 사용자가 '초안 작성하기' 버튼 클릭.
*   **Input Data:** 자소서 문항 제목, 글자 수 제한, 선택된 경험의 전체 구조화 데이터(STAR 기법 등), 채용 공고(JD) 주요 키워드.
*   **System Prompt (예시):**
    > "너는 15년 차 대기업 채용 전문 컨설턴트야. 제공된 지원자의 '경험 데이터'와 채용을 진행하는 기업의 '직무 공고 정보'를 융합해서, 입력된 자기소개서 문항에 대한 답변 초안을 작성해줘. 글자 수는 제한에 가깝게 맞추고, 두괄식 서술을 사용하여 핵심 역량이 돋보이도록 프로페셔널한 톤을 유지해."
*   **Output Format:** 자소서 본문 텍스트 + 필요시 두괄식 소제목.

### 3.3. 부분 편집 및 문장 톤 조정
*   **Trigger:** 사용자가 특정 문단을 드래그하고 `더 프로페셔널하게` 또는 `줄이기/늘리기` 툴바 클릭.
*   **Input Data:** 선택된 텍스트.
*   **System Prompt:**
    > "제공된 텍스트의 어조를 비즈니스에 맞는 전문가적인 톤으로 다듬되, 핵심 내용은 유지해."

## 4. 데이터 모델 (Data Model - DB 스키마 초안)
*   `CoverLetter` (자기소개서 테이블)
    *   `id` (PK)
    *   `userId` (FK, 작성자)
    *   `companyName` (String, "토스")
    *   `jobTitle` (String, "프론트엔드 엔지니어")
    *   `createdAt`, `updatedAt`
*   `CoverLetterItem` (자소서 문항별 테이블)
    *   `coverLetterId` (FK)
    *   `question` (String, 질문)
    *   `maxLength` (Int, 제한 글자 수)
    *   `content` (Text, 사용자가 수정한 최종 내용)
    *   `aiDraft` (Text, AI가 처음 생성해준 원본 초안)
    *   `selectedExperienceIds` (Array / JSON, [exp_id_1, exp_id_2])

## 5. UI/UX 요구사항 (UI/UX Requirements)
*   **Split View (화면 분할):** 한 화면에서 좌측은 '내 경험' 리스트나 '선택된 재료들'을 띄워두고, 우측은 타이핑을 치는 자소서 에디터로 나누어 배치. 화면 전환 없이도 재료를 보면서 글을 쓸 수 있게 편의성 제공.
*   **글자 수 카운터:** 에디터 하단에 항상 현재 글자 수와 공백 포함/제외된 글자 수를 색상 인디케이터로 노출.
*   **인라인 AI 툴바:** Notion AI처럼 텍스트를 드래그하면 팝업 형태의 툴바가 떠서 문장 교정, 톤 조절 등의 AI 액션을 바로 수행하도록 기획.
