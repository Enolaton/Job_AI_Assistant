[JobAI Assist PostgreSQL 운영 및 상세 가이드]

현재 서비스의 주 데이터베이스는 MongoDB에서 PostgreSQL로 전환되었습니다.
모든 데이터 관리는 Prisma(ORM)를 통해 수행됩니다.

1. 필수 도구 설치: pgAdmin 4
   - PostgreSQL의 실시간 데이터를 확인하고 관리하기 위한 GUI 도구입니다.

2. 환경 변수 설정 (.env 및 .env.local)
   - 프로젝트 최상위 경로에 .env 또는 .env.local 파일을 생성합니다.
   - 아래의 형식으로 DATABASE_URL을 작성해야 앱이 데이터베이스를 인식합니다.
   
   DATABASE_URL="postgresql://<유저네임>:<비밀번호>@localhost:5432/job_ai_database?schema=public"

3. Prisma 주요 명령어 (DB 관리)
   - `npx prisma db push`: schema.prisma의 내용을 실제 DB 테이블로 동기화합니다. (초기 단계 추천)
   - `npx prisma generate`: DB를 다루는 클라이언트 코드를 생성합니다. (스키마 변경 시 필수 실행)
   - `npx prisma studio`: 웹 브라우저 기반으로 데이터를 직접 확인하고 수정할 수 있는 관리 도구를 켭니다.

4. 프로젝트 소스 코드 연동 (lib/prisma.ts)
   - `PrismaClient`의 인스턴스를 하나만 유지하는 싱글톤(Singleton) 패턴이 적용되어 있습니다.
   - API나 서버 컴포넌트에서 `import prisma from "@/lib/prisma"`로 불러와서 사용하면 됩니다.

5. 트러블슈팅
   - 'Can't reach database server': PostgreSQL 서비스가 실행 중인지, 포트번호(5432)가 맞는지 확인하세요.
   - 'Authentication failed': 유저네임(`postgres`)과 비밀번호가 .env 파일과 일치하는지 확인하세요.
